Описание решения:
1. GeoData - структуры данных, методы поиска и работа с файлом гео-данных.
2. DataLoadTests - тестовое консольное приложение для профилирования/тестирования GeoData.
3. GeoSearch - проект web-приложения для поиска местоположения
    /index.html - пользовательская страница для поиска
    /test.html - страница для тестирования API

Некоторые вопросы/выводы:
1. Основное, на что прошу обратить внимание: я возможно, знаю не все особенности для решения данной задачи,
   но я могу решить любую задачу, даже если для нее решение неизвестно, используя системный подход аналитические навыки и опыт.

2. Есть куда еще оптимизировать (чтение элементов, сравнение строк...).

3. Для диапазонов ip-адресов количество индексных записей 12500 а не как в описании задачи 100000 (что на оставшейся части не посмотрел).

4. Тестирование api лучше запускать не в Chrome, не успел подправить - при большом количестве обновлений dom выдает ошибку err_insufficient_resources.

5. Из протестированных в консоли вариантов, наиболее эффективный по времени загрузки и скорости работы - копирование файла в поток в памяти.
